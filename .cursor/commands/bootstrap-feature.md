# Новая фича по PRD

1. Прочитай соответствующий файл в `prds/` (номер задачи пользователя или последний по смыслу).
2. Сверься с правилами `.cursor/rules/signal-lab-stack.mdc` — стек не менять без аргументации.
3. Составь минимальный план файлов (backend/frontend/prisma/compose).
4. Реализуй инкрементально; после значимых изменений предложи команды проверки из README или `verify-observability`.
